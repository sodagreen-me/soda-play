# SodaPlay - 歌曲数据构建指南

## 任务说明

为 SodaPlay 项目构建苏打绿的完整歌曲数据库。该数据为全局共享资源，供所有小游戏按需读取不同字段（如「苏打AI猜」读取 AI 描述，「歌词接龙」读取歌词等）。

## 数据结构

每首歌曲包含以下字段：

```ts
interface Song {
  name: string;         // 歌名
  artist: string;       // 演唱者（如「苏打绿」、「吴青峰」）
  album: string;        // 所属专辑
  lyrics: string;       // 完整歌词
  aiDescription: [AiDescription, AiDescription, AiDescription]; // 3 组 AI 描述与解释
}

interface AiDescription {
  text: string;        // AI 风格描述（用于「苏打AI猜」题面）
  explanation: string; // 简单解释（答题后展示，用于说明描述抓取的线索）
}
```

## AI 描述风格要求

描述模拟一个 AI「用它的方式理解一首歌」，核心原则：

每首歌必须提供 3 组 `aiDescription`，写成固定长度数组；每组包含 `text` 和 `explanation`。「苏打AI猜」每次出题会从该歌曲的 3 组描述中随机取 1 组，答题前展示 `text`，答题后展示对应的 `explanation`。

1. **不直接引用歌词原文**，也不能换一种方式描述歌词原文，但可以化用意象
2. **语气像一个 AI 在认真分析**——略带理性、呆板、甚至一本正经地胡说
3. **保留足够的辨识度**——资深歌迷读完后应该能联想到对应歌曲
4. **适当加入幽默感**——可以夸张、类比、用反差制造趣味
5. **1-2 句话为宜**，不要写太多，紧凑精炼
6. **不要出现歌名本身**（避免直接泄露答案），也不能换一种方式描述歌名
7. **用第一人称主观感受来写**——不要一直强调「歌手」如何，而是表达你对歌词的直接感受和理解
8. **中文文本标点规范**——提到专辑使用书名号《》，提到歌曲使用单书名号〈〉，引用内容使用竖引号「」；歌词原文不因标点规范而改写

### AI 解释要求

- 每条 `text` 必须有一条对应的 `explanation`
- `explanation` 用于答题后展示，必须直白说明 `text` 和歌曲的对应关系
- `explanation` 要简短，1 句话为宜，不要写成另一条谜面
- `explanation` 必须引用歌词原文，或点明非常具体的歌曲意象、叙事结构、编曲线索
- 如果 `text` 把某个具体事物抽象化，`explanation` 必须指出该事物是什么；例如 `text` 写「情绪故障日志」，`explanation` 要指出它对应「抽屉越来越拥挤」「来不及」等具体线索
- `explanation` 也用于复盘描述质量：如果解释无法说清对应关系，说明该 `text` 需要重写

### 风格参考示例

> 创作者坦言这是一首很简单的歌，简单到让他有点不好意思，试图用最朴素的方式描述一种感情。
> —— 〈小情歌〉

> 歌手声称某个人是天上天下独一无二的，比任何风景都好看，用宇宙级别的夸张手法来表达赞美。
> —— 〈无与伦比的美丽〉

> 整首歌在传达一个信息：非常想念某人。但偏偏嘴硬，说想念却「不让你知道」。
> —— 〈我好想你〉

> 一首关于迟到的歌，用极度的夸张来描述一种错过感，好像晚了一千年那么久。
> —— 〈迟到千年〉

> 以饮品命名的歌，似乎在用泡茶的过程来比喻某种感情——需要等待、需要耐心，最后才能品尝到花香四溢的美好。
> —— 〈花茶〉

### 避免的写法

- ❌ 直接复制歌词原句（如「这是一首小小情歌」）
- ❌ 过于抽象，无法定位到具体歌曲（如「一首关于爱情的歌」）
- ❌ 过于直白，毫无猜测乐趣（如「这首歌的名字里有个「情」字」）

## Prompt 模板（用于生成 AI 描述）

将以下 prompt 与完整歌词一起发送给 AI：

```
你是一个有趣的音乐分析 AI，正在为一个猜歌游戏生成歌曲描述。

规则：
- 你会用 AI 的视角来「解读」一首歌，语气理性、一本正经，但可以带有幽默感
- 不要直接引用歌词原文，也不能换一种方式描述歌词原文，但可以化用歌曲中的意象和关键词
- 描述中不要出现歌名本身，也不能换一种方式描述歌名
- 描述要保留足够辨识度，让熟悉这首歌的粉丝能猜出来
- 可以使用夸张、类比、反差等手法增加趣味性
- 控制在 1-2 句话，每句不要太长
- 每首歌生成 3 组描述与解释，三组需从不同角度切入，避免句式重复
- 每组必须包含 `text` 和 `explanation`，二者需要一一对应
- `text` 不要直接引用歌词；`explanation` 可以且应该引用短歌词或明确意象
- 用中文撰写

请阅读以下歌词，然后生成 3 组 AI 风格的歌曲描述与解释，输出为 TypeScript 对象数组。

歌名：【歌名】
专辑：【专辑名】

歌词：
【粘贴完整歌词】
```

## 工作流程

1. 确定要录入的专辑
2. 逐首准备：歌名 + 演唱者 + 专辑名 + 完整歌词
3. 使用上述 prompt 为每首歌生成 3 组 AI 描述与解释
4. 检查描述质量（辨识度、趣味性、是否泄露歌名）
5. 写入对应的专辑文件（如不存在则新建）
6. 如新建了专辑文件，在 `index.ts` 中添加 import 并汇总

## 注意事项

- 歌名和专辑名请确保准确
- 歌词使用多行模板字符串保存，保持原始换行；歌词正文每行不要额外缩进
- 建议每批处理 5-6 首，生成后人工检查质量再入库
- 最终数据量建议 30 首以上，以保证游戏有足够的题目池
- 新增专辑文件后，文件命名使用英文 kebab-case（如 `story-of-time.ts`）
