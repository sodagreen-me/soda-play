# SodaPlay - 歌曲数据构建指南

## 任务说明

为 SodaPlay 项目构建苏打绿的完整歌曲数据库。该数据为全局共享资源，供所有小游戏按需读取不同字段（如「苏打AI猜」读取 AI 描述，「歌词接龙」读取歌词等）。

## 数据结构

每首歌曲包含以下字段：

```ts
interface Song {
  name: string;         // 歌名
  artist: string;       // 演唱者（如"苏打绿"、"吴青峰"）
  album: string;        // 所属专辑
  lyrics: string;       // 完整歌词
  aiDescription: string; // AI 风格描述（用于「苏打AI猜」游戏）
}
```

## 文件组织

数据按专辑拆分为独立文件，通过索引文件汇总：

```
src/data/songs/
├── types.ts                # Song 接口定义
├── index.ts                # 汇总导出 allSongs

├── summer.ts               # 夏/狂热 (2009)
├── what-troubles-you.ts    # 你在烦恼什么 (2011)
├── autumn.ts               # 秋：故事 (2013)

```

## 每个专辑文件格式

```ts
import type { Song } from './types';

export const songs: Song[] = [
  {
    name: "歌名",
    artist: "苏打绿",
    album: "专辑名",
    lyrics: `完整歌词，第一行\n第二行\n...`,
    aiDescription: "AI 生成的描述",
  },
  // ...
];
```

## AI 描述风格要求

描述模拟一个 AI "用它的方式理解一首歌"，核心原则：

1. **不直接引用歌词原文**，但可以化用意象
2. **语气像一个 AI 在认真分析**——略带理性、呆板、甚至一本正经地胡说
3. **保留足够的辨识度**——资深歌迷读完后应该能联想到对应歌曲
4. **适当加入幽默感**——可以夸张、类比、用反差制造趣味
5. **2-4 句话为宜**，不要太长
6. **不要出现歌名本身**（避免直接泄露答案）

### 风格参考示例

> 创作者坦言这是一首很简单的歌，简单到让他有点不好意思。歌词反复强调「简单」二字，试图用最朴素的方式描述一种感情。他说这是一首小小的、简单的歌。
> —— 小情歌

> 歌手声称某个人是天上天下独一无二的，比任何风景都好看。用宇宙级别的夸张手法来表达赞美——天上风筝在天上飞，地上人儿在地上追。
> —— 无与伦比的美丽

> 整首歌在传达一个信息：非常想念某人。但歌手偏偏嘴硬，说想念却「不让你知道」，还称之为「多完美」的默契。典型的口是心非。
> —— 我好想你

> 一首关于迟到的歌，但迟到的时间单位不是分钟，而是千年。歌手用极度的夸张来描述一种错过感，好像晚了一千年那么久。
> —— 迟到千年

> 以饮品命名的歌。歌手似乎在用泡茶的过程来比喻某种感情——需要等待、需要耐心，最后才能品尝到花香四溢的美好。
> —— 花茶

### 避免的写法

- ❌ 直接复制歌词原句（如"这是一首小小情歌"）
- ❌ 过于抽象，无法定位到具体歌曲（如"一首关于爱情的歌"）
- ❌ 过于直白，毫无猜测乐趣（如"这首歌的名字里有个'情'字"）

## Prompt 模板（用于生成 AI 描述）

将以下 prompt 与完整歌词一起发送给 AI：

```
你是一个有趣的音乐分析 AI，正在为一个猜歌游戏生成歌曲描述。

规则：
- 你会用 AI 的视角来"解读"一首歌，语气理性、一本正经，但可以带有幽默感
- 不要直接引用歌词原文，但可以化用歌曲中的意象和关键词
- 描述中不要出现歌名本身
- 描述要保留足够辨识度，让熟悉这首歌的粉丝能猜出来
- 可以使用夸张、类比、反差等手法增加趣味性
- 控制在 2-4 句话，每句不要太长
- 用中文撰写

请阅读以下歌词，然后生成一段 AI 风格的歌曲描述。

歌名：【歌名】
专辑：【专辑名】

歌词：
【粘贴完整歌词】
```

## 工作流程

1. 确定要录入的专辑
2. 逐首准备：歌名 + 演唱者 + 专辑名 + 完整歌词
3. 使用上述 prompt 为每首歌生成 AI 描述
4. 检查描述质量（辨识度、趣味性、是否泄露歌名）
5. 写入对应的专辑文件（如不存在则新建）
6. 如新建了专辑文件，在 `index.ts` 中添加 import 并汇总

## 注意事项

- 歌名和专辑名请确保准确
- 歌词建议使用换行符 `\n` 分隔
- 建议每批处理 5-6 首，生成后人工检查质量再入库
- 最终数据量建议 30 首以上，以保证游戏有足够的题目池
- 新增专辑文件后，文件命名使用英文 kebab-case（如 `story-of-time.ts`）
